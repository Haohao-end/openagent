❌ 原始错误

v-model cannot be used on a prop, because local prop bindings are not writable.

错误原因：在 FullscreenCodeEditor.vue 中直接对 prop 使用 v-model:visible， 但 Vue 3 中 props 是只读的，不能直接修改。

✅ 解决方案

创建本地响应式变量，通过 watch 同步 props 和 emits

// ❌ 错误的写法 const props = defineProps({ visible: Boolean }) <a-modal v-model:visible="visible" /> // 试图修改 prop！ // ✅ 正确的写法 const localVisible = ref(props.visible) // 创建本地变量 watch(() => props.visible, (v) => localVisible.value = v) // 监听 prop watch(localVisible, (v) => emits('update:visible', v)) // 同步到外部 <a-modal v-model:visible="localVisible" /> // 使用本地变量

📝 修改清单

  • 1. 添加本地状态：const localVisible = ref(props.visible)
  • 2. 监听 prop 变化：watch(() => props.visible, ...)
  • 3. 监听本地变化：watch(localVisible, ...)
  • 4. 更新模板：v-model:visible="localVisible"
  • 5. 同样处理 modelValue：创建 localCode 并同步

💡 核心概念

在 Vue 3 中，props 是单向数据流，子组件不能直接修改 props。 如果需要在子组件中修改值，应该：

  1. 创建本地响应式变量（ref/reactive）
  2. 通过 watch 监听 prop 变化，更新本地变量
  3. 通过 emit 事件通知父组件更新 prop
  4. 在模板中使用本地变量而不是 prop
